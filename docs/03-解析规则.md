# 解析规则

本文档详细说明 RuleFlow 如何解析各种代理协议和订阅格式。

## 支持的协议

| 协议 | URL Scheme | 说明 |
|------|-----------|------|
| Trojan | `trojan://` | TLS 隧道代理 |
| VMess | `vmess://` | V2Ray 协议 |
| VLESS | `vless://` | V2Ray 轻量级协议 |
| Shadowsocks | `ss://` | SOCKS5 加密代理 |
| Hysteria2 | `hysteria2://` / `hy2://` | QUIC -based 代理 |
| TUIC | `tuic://` | QUIC -based 代理 |
| AnyTLS | `anytls://` | 通用 TLS 隧道 |

## 订阅内容解析流程

### 整体流程

```
原始订阅内容
    ↓
1. 检查是否为 HTML (错误页面)
    ↓
2. 检测是否为 Clash YAML (含 proxies: 字段)
    ├─ 是 → 直接解析 YAML
    └─ 否 → 继续
    ↓
3. 检查是否包含协议 URL
    ├─ 是 → 提取 URL
    └─ 否 → 尝试 Base64 解码
    ↓
4. 解析每个协议 URL 为 ProxyNode
    ↓
5. 去重
    ↓
返回节点列表
```

### 代码入口

```go
// internal/app/parser.go:906
func ParseSubscription(content string) ([]*ProxyNode, error)
```

## 各协议解析规则

### 1. Trojan 协议

**URL 格式**:
```
trojan://password@server:port?query=value#name
```

**解析字段**:

| 字段 | 来源 | 说明 |
|------|------|------|
| `password` | URL Userinfo | 密码 |
| `server` | URL Hostname | 服务器地址 |
| `port` | URL Port | 端口，默认 443 |
| `name` | `name` / `hash` query / fragment | 节点名称 |
| `sni` | `sni` query | TLS SNI，默认 server |
| `allowInsecure` | `allowInsecure` query | 是否跳过证书验证 |
| `alpn` | `alpn` query | ALPN 协议，逗号分隔 |
| `type` / `path` / `host` | query | WebSocket 传输配置 |

**示例**:
```
trojan://mypassword@example.com:443?name=🇺🇸US&sni=example.com#美国节点
```

---

### 2. VMess 协议

**URL 格式**:
```
vmess://<base64-encoded-json>
```

**JSON 结构**:
```json
{
  "id": "uuid",
  "add": "server",
  "port": 443,
  "ps": "节点名称",
  "aid": 0,
  "net": "tcp",
  "tls": "tls",
  "sni": "example.com",
  "path": "/ws",
  "host": "example.com",
  "alpn": "h2,http/1.1"
}
```

**解析字段**:

| 字段 | JSON Key | 说明 |
|------|----------|------|
| `uuid` | `id` | 用户 ID |
| `server` | `add` | 服务器地址 |
| `port` | `port` | 端口 |
| `name` | `ps` | 节点名称 |
| `alterID` | `aid` | Alter ID |
| `network` | `net` | 传输协议 (tcp/ws/grpc) |
| `tls` | `tls` | 是否启用 TLS |
| `sni` | `sni` | TLS SNI |
| `path` | `path` | WebSocket 路径 |
| `host` | `host` | WebSocket Host |

**示例**:
```
vmess://eyJhZGQiOiJleGFtcGxlLmNvbSIsInBvcnQiOjQ0MywiaWQiOiJ1dWlkIiwicHMiOiLwn6Wg5Y+WIn0=
```

---

### 3. VLESS 协议

**URL 格式**:
```
vless://uuid@server:port?query=value#name
```

**解析字段**:

| 字段 | 来源 | 说明 |
|------|------|------|
| `uuid` | URL Userinfo | 用户 ID |
| `server` | URL Hostname | 服务器地址 |
| `port` | URL Port | 端口，默认 443 |
| `name` | URL fragment | 节点名称 |
| `type` | `type` query | 传输协议 (tcp/ws/grpc) |
| `security` | `security` query | 安全类型 (tls/reality/none) |
| `sni` | `sni` query | TLS SNI |
| `fp` | `fp` query | uTLS 指纹 |
| `alpn` | `alpn` query | ALPN 协议 |
| `flow` | `flow` query | 流控 (xtls-rprx-vision) |
| `pbk` / `sid` | query | Reality 公钥/短 ID |

**Reality 配置**:
当 `security=reality` 时，额外解析:
- `pbk`: Reality Public Key
- `sid`: Reality Short ID

**示例**:
```
vless://uuid@example.com:443?security=reality&pbk=xxx&sid=yyy#🇯🇵JP
```

---

### 4. Shadowsocks 协议

**URL 格式** (两种):

格式 A (推荐):
```
ss://<base64(cipher:password)>@server:port#name
```

格式 B:
```
ss://cipher:password@server:port#name
```

**解析字段**:

| 字段 | 来源 | 说明 |
|------|------|------|
| `cipher` | Userinfo 部分 | 加密方法 |
| `password` | Userinfo 部分 | 密码 |
| `server` | Hostname | 服务器地址 |
| `port` | Port | 端口，默认 8388 |
| `name` | fragment | 节点名称 |

**支持的加密方法**:
- `aes-256-gcm`
- `chacha20-poly1305`
- `aes-128-gcm`
- 等...

**示例**:
```
ss://YWVzLTI1Ni1nY206cGFzc3dvcmQ=@example.com:8388#🇸🇬SG
```

## Clash YAML 订阅解析

除了协议 URL，RuleFlow 也支持直接解析 Clash 格式的 YAML 订阅。

**检测条件**: 内容包含 `proxies:` 字段

**解析流程**:
```
YAML 内容
    ↓
yaml.Unmarshal → 提取 proxies 数组
    ↓
遍历每个 proxy:
    ├─ 标准化 type (trojan/vmess/vless/ss/wireguard/hysteria2/tuic)
    ├─ 提取 name/server/port
    ├─ 其余字段放入 Options
    └─ 构造 ProxyNode
    ↓
返回节点列表
```

**支持的协议类型**:
- `trojan`
- `vmess`
- `vless`
- `ss` (Shadowsocks)
- `wireguard`
- `hysteria2` / `hy2`
- `tuic`

**示例 YAML**:
```yaml
proxies:
  - name: "🇺🇸 US-01"
    type: trojan
    server: example.com
    port: 443
    password: mypass
    sni: example.com
```

## Base64 解码策略

当订阅内容未直接包含协议 URL 时，尝试多种 Base64 解码方式:

### 解码尝试顺序

1. **整体 Base64 解码**
   - `StdEncoding`
   - `RawStdEncoding`
   - `URLEncoding`
   - `RawURLEncoding`
   - 自动补全 padding (`=`)

2. **逐行 Base64 解码**
   - 对每一行单独尝试解码
   - 成功行数 > 0 则使用

### Base64 清理

解码前先清理输入:
- 移除 ANSI 转义序列 (`\x1b[...m`)
- 保留字符: `A-Z`, `a-z`, `0-9`, `+`, `/`, `=`, `-`, `_`
- URL 安全 Base64 转换: `-` → `+`, `_` → `/`

## 节点去重

解析完成后对节点进行去重:

```go
// 去重逻辑
seen := make(map[string]struct{})
for _, url := range urls {
    if _, ok := seen[url]; !ok {
        seen[url] = struct{}{}
        result = append(result, url)
    }
}
```

**去重键**: 原始协议 URL 字符串

## 错误处理

### 常见错误

| 错误 | 说明 |
|------|------|
| `订阅服务器返回了 HTML 页面` | 检测到 `<!doctype html>` 或 `<html`，可能订阅地址无效 |
| `订阅服务器返回了空内容` | 内容为空 |
| `未找到有效的代理链接` | 未识别出任何协议 URL |
| `未能成功解析任何节点` | 所有 URL 解析失败 |

### 容错策略

- 单个 URL 解析失败 → 记录错误，继续下一个
- 至少成功解析一个节点 → 返回成功结果 + 记录失败日志
- 全部失败 → 返回错误，包含第一个失败原因

## 辅助函数

### 正则表达式模式

```go
// 协议 URL 匹配
trojanURLPattern    = regexp.MustCompile(`trojan://[^\s]+`)
vmessURLPattern     = regexp.MustCompile(`vmess://[a-zA-Z0-9_=]+`)
vlessURLPattern     = regexp.MustCompile(`vless://[^\s]+`)
ssURLPattern        = regexp.MustCompile(`ss://[^\s]+`)
hysteria2URLPattern = regexp.MustCompile(`(?:hysteria2|hy2)://[^\s]+`)
tuicURLPattern      = regexp.MustCompile(`tuic://[^\s]+`)
anytlsURLPattern    = regexp.MustCompile(`anytls://[^\s]+`)
allProtocolPattern  = regexp.MustCompile(`(trojan|vmess|vless|ss|anytls|hysteria2|hy2|tuic)://[^\s]+`)
```

### 端口解析

```go
func parsePortWithDefault(portStr string, defaultPort int) (int, error)
```

- 空字符串 → 返回默认端口
- 解析失败 → 返回默认端口 + 错误

### 查询参数布尔值

```go
func queryBool(query url.Values, keys ...string) bool
```

- 按顺序尝试多个 key
- `"1"` 或 `"true"` (不区分大小写) → `true`
