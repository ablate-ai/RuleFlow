# 项目目录结构

本文档详细介绍 RuleFlow 项目的文件组织和目录结构。

## 整体结构

```
RuleFlow/
├── api/                          # HTTP 处理层
├── cache/                        # 缓存层（Redis）
├── config/                       # 配置加载
├── database/                     # 数据访问层（PostgreSQL）
├── deploy/                       # 部署配置
├── docs/                         # 项目文档
├── internal/app/                 # 核心业务逻辑
├── migrations/                   # 数据库迁移脚本
├── rules/                        # 内置规则模板
├── services/                     # 服务层
├── web/                          # Web 控制台前端
├── .env.example                  # 环境变量示例
├── Dockerfile                    # Docker 构建文件
├── Makefile                      # 构建脚本
├── main.go                       # 程序入口
└── go.mod/go.sum                 # Go 模块管理
```

## 目录详解

### api/ - HTTP 处理层

负责处理所有 HTTP 请求，包括 REST API 和 Web 界面路由。

| 文件 | 说明 |
|------|------|
| `handlers.go` | REST API 处理器，实现所有 CRUD 接口 |
| `middleware.go` | 中间件：鉴权、CORS、日志、恢复 |
| `response.go` | 统一响应格式定义 |
| `surge_managed_config.go` | Surge 托管配置支持 |

### cache/ - 缓存层

提供 Redis 缓存功能，用于缓存生成的配置文件。

| 文件 | 说明 |
|------|------|
| `redis.go` | Redis 客户端初始化和基础操作 |
| `subscription_cache.go` | 订阅配置缓存管理 |

### config/ - 配置加载

负责从环境变量加载配置。

| 文件 | 说明 |
|------|------|
| `config.go` | 配置结构定义和加载逻辑 |

### database/ - 数据访问层

PostgreSQL 数据库操作层，使用 Repository 模式。

| 文件 | 说明 |
|------|------|
| `database.go` | 数据库连接初始化 |
| `id.go` | ID 生成器（雪花算法） |
| `subscription_repo.go` | 订阅源数据访问 |
| `node_repo.go` | 节点数据访问 |
| `template_repo.go` | 模板数据访问 |
| `config_policy_repo.go` | 配置策略数据访问 |
| `rule_source_repo.go` | 规则源数据访问 |
| `config_access_log_repo.go` | 配置访问日志数据访问 |

### internal/app/ - 核心业务逻辑

项目的核心模块，包含协议解析、配置生成等核心功能。

| 文件 | 说明 |
|------|------|
| `models.go` | 数据模型定义（ProxyNode、Proxy、Group 等） |
| `parser.go` | 多协议节点 URL 解析器 |
| `config_builder.go` | Clash Meta / Stash YAML 配置生成器 |
| `surge_builder.go` | Surge INI 配置生成器 |
| `singbox_builder.go` | Sing-Box 配置生成器 |
| `subscription.go` | 订阅拉取逻辑 |
| `country_emoji.go` | 国家/地区 Emoji 自动添加 |
| `rule_set.go` | 规则集处理 |
| `wireguard.go` | WireGuard 协议支持 |
| `handlers.go` | 内部处理函数 |

### services/ - 服务层

业务逻辑编排层，协调各 Repository 完成复杂业务操作。

| 文件 | 说明 |
|------|------|
| `subscription_service.go` | 订阅源服务 |
| `node_service.go` | 节点服务 |
| `template_service.go` | 模板服务 |
| `config_policy_service.go` | 配置策略服务 |
| `rule_source_service.go` | 规则源服务 |
| `subscription_sync_service.go` | 订阅同步服务 |
| `rule_source_sync_service.go` | 规则源同步服务 |
| `subscription_scheduler.go` | 订阅自动刷新调度器 |
| `rule_source_scheduler.go` | 规则源自动刷新调度器 |
| `log_cleanup_scheduler.go` | 日志清理调度器 |
| `maintenance_service.go` | 维护服务 |

### rules/ - 内置规则模板

| 文件 | 说明 |
|------|------|
| `clash.yaml` | Clash Meta / Stash 默认模板 |
| `surge.conf` | Surge 默认模板 |

### web/ - Web 控制台前端

纯 HTML/JavaScript 前端，无需构建工具。

| 文件 | 说明 |
|------|------|
| `index.html` | 仪表盘首页 |
| `login.html` | 登录页面 |
| `subscriptions.html` | 订阅源管理 |
| `nodes.html` | 节点管理 |
| `templates.html` | 模板管理 |
| `configs.html` | 配置策略管理 |
| `rule_sources.html` | 规则源管理 |
| `config_access_logs.html` | 访问日志 |

### migrations/ - 数据库迁移

| 文件 | 说明 |
|------|------|
| `init.sql` | 数据库初始化脚本 |

### deploy/ - 部署配置

| 文件 | 说明 |
|------|------|
| `docker-compose.yaml` | Docker Compose 部署配置 |

## 依赖关系

```
main.go (入口)
    ├── api/ (HTTP 层)
    │   └── services/ (服务层)
    │       ├── database/ (数据层)
    │       └── internal/app/ (核心逻辑)
    ├── cache/ (缓存层)
    └── config/ (配置)
```

## 数据流向

1. **HTTP 请求** → `api/handlers.go`
2. **Handler** → `services/*_service.go`
3. **Service** → `database/*_repo.go` (数据读写)
4. **Service** → `internal/app/*.go` (业务逻辑)
5. **返回响应** → `api/response.go`
